# 🏴 Summary

## 1.2 스프링 생태계

스프링은 하나의 프레임워크가 아니라 여러 모듈로 구성된 생태계입니다.

- **Spring Core**
  - IoC와 DI를 기반으로 객체를 생성하고 관리하는 핵심 모듈
  - `ApplicationContext`를 통해 객체의 생명주기를 관리

- **Spring MVC**
  - HTTP 요청을 처리하는 웹 애플리케이션 개발을 지원

- **Spring Data Access**
  - 데이터베이스 접근과 영속성 계층 구현을 위한 기능 제공

- **Spring Testing**
  - 단위 테스트와 통합 테스트를 지원하는 테스트 도구 제공

### Spring Boot

스프링을 더 쉽게 사용할 수 있도록 기본 설정(Auto Configuration)을 제공하여 개발자가 필요한 설정을 최소화할 수 있도록 돕습니다.

---

## 1.4 프레임워크를 사용하지 말아야 할 때

다음과 같은 경우에는 프레임워크 사용이 적합하지 않을 수 있습니다.

- 애플리케이션 규모가 매우 작은 경우 (예: 서버리스 함수)
- 보안 정책상 외부 프레임워크 사용이 제한되는 경우
- 과도한 커스터마이징이 필요한 경우
- 기존 시스템을 프레임워크로 변경해도 실질적인 이점이 없는 경우


# ✔️ Key Points

- 스프링은 하나의 프레임워크가 아니라 여러 모듈로 구성된 생태계입니다.
- IoC는 객체 생성과 제어를 프레임워크가 담당하는 스프링의 핵심 개념입니다.
- **Spring Boot**는 새로운 프레임워크가 아니라 스프링의 기본 설정을 자동화해 개발 편의성을 높여줍니다.
- 프레임워크는 항상 정답이 아니며, 프로젝트의 규모와 목적에 따라 적절한 선택이 중요합니다.

# 👀 In Practice
아직 직접적인 실무 적용 사례는 없지만, 앞으로 Spring Core와 IoC를 학습하면서 Go에서의 의존성 관리 방식과 비교해보고 싶습니다.

# 🖤 Thought

스프링을 하나의 프레임워크라고만 생각했지만, 실제로는 Spring Core, MVC, Data, Testing 등 여러 모듈을 아우르는 생태계라는 것을 알게 되었습니다.

또한 Spring Boot가 Spring을 대체하는 별도의 프레임워크가 아니라, 스프링을 더 쉽게 사용할 수 있도록 기본 설정을 제공하는 도구라는 점도 새롭게 이해하게 되었습니다.

무엇보다 스프링이 단순히 백엔드 웹 개발에만 사용되는 프레임워크가 아니라, 데이터 접근과 테스트 등 애플리케이션 전반을 지원하는 다양한 기능을 제공한다는 점이 인상 깊었습니다.
